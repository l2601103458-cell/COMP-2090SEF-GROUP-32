# Food Delivery System - Task 1 Introduction

## Main Idea
We are planning to make a food delivery system, which could be more convenient for people to order food delivery. The program of the system will be separated into four parts, customers, restaurants, delivery platform, and main system. The main functions of these parts are:
- Customers: account management, food ordering
- Restaurants: food order management, food menu management
- Delivery platform: system pages management, connections of restaurants and customers
- Main system: to maintain the whole system working, connections of all categories

## Programming Techniques
### Modular Programming
Modular programming will be used in connecting different functions and programs and combines into a main system. With modular programming, programs can be separated into different files and the categorization can become obvious. It can make the program code become easier to read and write. It will not affect the smoothness of program operation, but having a more convenient way of program building.

### Python Class
Python class is a way of building objects with functions and methods that can increase convenience in programming. It can break down complex structures into easier ways to read and understand. It increases the code reusability, and it can also help to handle more detailed aspects of each category.

## System Functions
In the system, users can create their own accounts, and each account has their own money saved. Users can order food from different restaurants, by using the money in their accounts. Users can add money into the account for more consumption.
